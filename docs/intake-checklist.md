# Dashboard Intake Checklist

Use this before adding a dashboard to the curated catalog.

## Permission And Attribution

- [ ] Original creator is known or clearly marked unknown.
- [ ] Permission/status is recorded.
- [ ] Original source is linked or described when known.
- [ ] Curator/packager is recorded.
- [ ] Material changes are documented.

## Safety

- [ ] No customer names.
- [ ] No hostnames or internal URLs.
- [ ] No tokens, access IDs, secrets, or credentials.
- [ ] No private ticket numbers or private repo paths.
- [ ] Screenshots are sanitized.

## Dashboard Quality

- [ ] Dashboard imports into a test portal.
- [ ] Description matches what the dashboard actually contains.
- [ ] Required data sources, properties, or widget behavior are documented.
- [ ] Widgets render without broken layout.
- [ ] Dashboard name is clear and customer-safe.

## Catalog Fit

- [ ] Category is meaningful.
- [ ] Tags add information beyond the dashboard name.
- [ ] `source_path` gives a clean import folder path.
- [ ] Attribution is visible in `DASHBOARD.md`.
- [ ] `npm run build` passes.
- [ ] `npm run validate` passes.
